# Ёлы-Палы — веб-сайт базы отдыха

Дипломный проект: веб-сайт для базы отдыха с каталогом размещения, галереей, ценами, формой предварительного бронирования и панелью администратора.

## Что реализовано

- Главная страница сайта.
- Раздел «О базе отдыха».
- Каталог вариантов размещения: Домик, Люкс, Полулюкс, Стандарт.
- Раздел «Цены» с таблицей тарифов.
- Раздел услуг.
- Галерея с просмотром фотографий.
- Карта местоположения и отзывы.
- Форма предварительного бронирования.
- Автоматический расчёт стоимости по датам, количеству гостей и дополнительным услугам.
- Проверка занятости дат перед отправкой заявки.
- Сохранение заявок в SQLite-базу данных.
- Улучшенная панель администратора: статистика, поиск, фильтры по статусу, размещению и датам.
- Управление заявками: подтверждение, отклонение, возврат в новые и удаление.
- Управление вариантами размещения: название, описание, фото, вместимость, удобства и цены.
- Загрузка фотографий через админ-панель и вывод изображений в галерею сайта.
- Управление дополнительными услугами: название, описание, иконка, стоимость и участие в форме бронирования.
- Блок ближайших заездов и популярности вариантов размещения.
- Экспорт заявок в CSV с учётом выбранных фильтров.
- Защита форм с помощью CSRF-токена.
- Согласие пользователя на обработку персональных данных.

## Запуск проекта

1. Установить Python 3.10 или выше.
2. Открыть папку проекта в терминале.
3. Создать виртуальное окружение:

```bash
python -m venv venv
```

4. Активировать виртуальное окружение.

Для Windows:

```bash
venv\Scripts\activate
```

Для macOS/Linux:

```bash
source venv/bin/activate
```

5. Установить зависимости:

```bash
pip install -r requirements.txt
```

6. Запустить приложение:

```bash
python app.py
```

7. Открыть сайт в браузере:

```text
http://127.0.0.1:5000
```

## Панель администратора

Адрес панели:

```text
http://127.0.0.1:5000/admin
```

Данные администратора создаются при первом запуске приложения. Их можно задать через переменные окружения `ADMIN_USERNAME` и `ADMIN_PASSWORD` или изменить в файле `app.py` до первого запуска.

В панели администратора доступны:

- карточки статистики по всем заявкам;
- поиск по имени, телефону, комментарию и выбранному размещению;
- фильтр по статусу заявки;
- фильтр по варианту размещения;
- фильтр по датам заезда;
- быстрые кнопки смены статуса;
- просмотр ближайших подтверждённых заездов;
- статистика популярности домиков и номеров;
- экспорт заявок в CSV;
- редактирование домиков и номеров;
- изменение цен проживания;
- добавление и редактирование дополнительных услуг;
- загрузка новых фотографий для карточек размещения и галереи;
- управление галереей клиентской страницы;
- скрытие неактуальных вариантов размещения с клиентской страницы.

В реальном использовании пароль нужно заменить через переменные окружения:

```bash
set ADMIN_USERNAME=Olesya
set ADMIN_PASSWORD=your_password
set SECRET_KEY=your_secret_key
```

Для PowerShell:

```powershell
$env:ADMIN_USERNAME="Olesya"
$env:ADMIN_PASSWORD="your_password"
$env:SECRET_KEY="your_secret_key"
```

## База данных

При первом запуске приложение автоматически создаёт файл:

```text
data/site.db
```

В базе данных создаются таблицы:

- `rooms` — варианты размещения и цены;
- `services` — дополнительные услуги и их стоимость;
- `gallery_photos` — фотографии галереи сайта;
- `bookings` — заявки на бронирование;
- `admins` — учётные записи администраторов.

## Примечание для дипломной работы

Система реализует предварительное бронирование. Окончательное подтверждение заявки выполняет администратор через панель управления. Это позволяет избежать автоматического подтверждения спорных броней и оставить контроль за сотрудником базы отдыха.
