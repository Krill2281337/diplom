# Структура базы данных

В проекте используется SQLite. База данных создаётся автоматически при первом запуске приложения и хранится в файле `data/site.db`.

## Таблица `rooms`

Назначение: хранение информации о вариантах размещения.

| Поле | Тип | Описание |
|---|---|---|
| `name` | TEXT PRIMARY KEY | Название варианта размещения |
| `capacity` | INTEGER | Максимальное количество гостей |
| `weekday_price` | INTEGER | Цена за сутки в будние дни |
| `weekend_price` | INTEGER | Цена за сутки в выходные дни |
| `description` | TEXT | Краткое описание варианта |
| `amenities` | TEXT | Перечень удобств |
| `image` | TEXT | Путь к изображению |
| `badge` | TEXT | Метка карточки на сайте |
| `sort_order` | INTEGER | Порядок отображения на сайте |
| `is_active` | INTEGER | Признак отображения на клиентской странице |


## Таблица `services`

Назначение: хранение дополнительных услуг, которые отображаются на сайте и могут участвовать в расчёте заявки.

| Поле | Тип | Описание |
|---|---|---|
| `code` | TEXT PRIMARY KEY | Технический код услуги для формы бронирования |
| `name` | TEXT | Название услуги |
| `description` | TEXT | Описание услуги для клиентской страницы |
| `price` | INTEGER | Стоимость услуги в рублях |
| `price_type` | TEXT | Тип расчёта: фиксированная, за гостя/ночь, бесплатно, по согласованию |
| `icon` | TEXT | CSS-класс иконки Font Awesome |
| `display_on_site` | INTEGER | Показывать услугу в разделе «Услуги» |
| `available_in_booking` | INTEGER | Показывать услугу в форме бронирования |
| `sort_order` | INTEGER | Порядок отображения |
| `is_active` | INTEGER | Признак активности услуги |


## Таблица `gallery_photos`

Назначение: хранение фотографий, которые выводятся в галерее на клиентской странице сайта.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Идентификатор фотографии |
| `title` | TEXT | Название фотографии для отображения на сайте |
| `image` | TEXT UNIQUE | Путь к файлу изображения в папке `static/img` или `static/uploads` |
| `sort_order` | INTEGER | Порядок отображения в галерее |
| `is_active` | INTEGER | Признак отображения фотографии на клиентской странице |

## Таблица `bookings`

Назначение: хранение заявок на предварительное бронирование.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Идентификатор заявки |
| `created_at` | TEXT | Дата и время создания заявки |
| `guest_name` | TEXT | Имя клиента |
| `phone` | TEXT | Контактный телефон |
| `cottage` | TEXT | Выбранный вариант размещения |
| `check_in` | TEXT | Дата заезда |
| `check_out` | TEXT | Дата выезда |
| `nights` | INTEGER | Количество ночей |
| `guests` | INTEGER | Количество гостей |
| `extras` | TEXT | Дополнительные услуги |
| `comment` | TEXT | Комментарий клиента |
| `total_price` | INTEGER | Предварительная стоимость |
| `status` | TEXT | Статус заявки: Новая, Подтверждена, Отклонена |

## Таблица `admins`

Назначение: хранение учётных данных администраторов.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT | Идентификатор администратора |
| `username` | TEXT UNIQUE | Логин администратора |
| `password_hash` | TEXT | Хэш пароля |

## Управление данными из админ-панели

Администратор может менять данные таблиц `rooms`, `services` и `gallery_photos` через страницу `/admin/catalog`. После сохранения изменения сразу отображаются на клиентской странице сайта и используются при расчёте стоимости бронирования.

## Проверка занятости дат

При отправке заявки система проверяет, есть ли пересечение выбранного диапазона дат с заявками в статусе `Новая` или `Подтверждена`.

Логика пересечения:

```text
существующая_дата_заезда < новая_дата_выезда
и
существующая_дата_выезда > новая_дата_заезда
```

Если пересечение найдено, новая заявка не сохраняется, а пользователь получает сообщение о занятости дат.


## Загрузка изображений

Загруженные через админ-панель фотографии сохраняются в папку `static/uploads`. После загрузки файл можно выбрать в карточке домика или номера, а также добавить в галерею сайта. Это позволяет обновлять визуальный контент без ручного редактирования HTML-кода.
